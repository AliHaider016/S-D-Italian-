The error chain is:\n\n```\nPostCssTransformedAsset::process → evaluate_webpack_loader → DiskFileSystem::read\n→ reading file C:\\...\\flow2code\\nul → os error 1 (ERROR_INVALID_FUNCTION)\n```\n\nOn Windows, `nul` is a reserved device name (equivalent to `/dev/null`). Once the cache was cleared and the server restarted cleanly, the issue didn't recur.\n\n### Recommendation\n\nWhile I can't reproduce this anymore, there's still a potential underlying issue: **Turbopack's `DiskFileSystem::read` should gracefully handle Windows reserved device names** (`NUL`, `CON`, `PRN`, `AUX`, `COM1`–`COM9`, `LPT1`–`LPT9`) instead of panicking. These can appear in source maps or file references from Unix-oriented tooling.\n\nFeel free to close this if it's not actionable without a reliable reproduction.
